快速上手¶

这是给那些想要快速上手和运行的人。

  1. Docutils requires Python, available from http://www.python.org/.

    详情请参阅下面的 需求 。

  2. 使用 pip 从 PyPI 安装最新的稳定版本

    python -m pip install docutils

    要安装预发行版，附加选项 --pre。要安装 开发版本，请遵循下面 安装 部分中的说明。

  3. 使用 front-end scripts 转换 reStructuredText 文档。可以尝试

    rst2html.py FAQ.txt FAQ.html         (Unix)
python tools/rst2html.py FAQ.txt FAQ.html  (Windows)

    阅读 用法 了解细节。

目标¶

Docutils 项目的目标是创建一组工具，用于将纯文本文档处理成有用的格式，如 HTML、LaTeX、troff（手册页）、OpenOffice 和本地 XML。已经实施了对以下来源的支持：

  • 独立的文件。

  • PEPs (Python Enhancement Proposals).

计划支持以下资源：

  • 来自 Python 模块和包的内联文档，使用命名空间上下文提取

  • Email （RFC-822 标题，引用的摘录，签名，MIME 部分）

  • Wikis 与“维基链接”的全局参考查找

  • 复合文档，例如多个章节文件合并到一本书中。

  • 和其他发现的。

需求¶

要运行代码，必须安装 Python。

  • Docutils 0.16 及更高版本原生支持 Python 2.7 和3.5+。1

  • Docutils 0.14 放弃了对 2.4, 2.5, 3.1 和 3.2 的支持。

  • Docutils 0.10 放弃了对 Python 2.3 的支持。

  • 从 0.6 版本开始，Docutils 就与 Python 3 兼容。1

  • 对于 Docutils 0.5, Python 2.2.1+ 是必需的。

  • 在 Docutils 0.4 之前，Python 2.1 是必需的。

1(1,2)

直到 0.15 版本，Doctils 代码库都是使用 2to3 工具“按需”翻译的。

建议¶

如果安装了 Docutils，它会使用以下软件包来增强功能：

  • 安装 ：安装通常使用 pip 或 setuptools 完成。

  • PIL（Python Imaging Library）用于一些图像操作。

  • Pygments 包提供了“代码”指令和角色的语法高亮显示。

  • recommonmark 解析器用于解析 Markdown 格式的输入。

Docutils 链接列表 记录了 Docutils 和 reStructuredText 用户可能会觉得有用的项目。

开发版本¶

虽然我们试图遵循“早发布和经常发布”的政策，但新功能还是经常添加。由于 存储库 中的代码通常处于无 bug 状态，所以我们建议使用当前快照或工作副本。

要获得 snapshot，请转到代码页并单击下载 snapshot 按钮：

  • Docutils 代码、文档、前端工具和测试：https://sourceforge.net/p/docutils/code/HEAD/tree/trunk/docutils/

  • Sandbox （实验性的贡献代码）：https://sourceforge.net/p/docutils/code/HEAD/tree/trunk/sandbox/

要跟上最新的开发，请定期下载快照的新副本或使用 Docutils 代码库的工作副本。

继续下面的 安装 说明。

安装¶

从源代码安装 Docutils 的步骤：

  • 进入包含 setup.py 文件的目录：

    snapshot 必须首先解压到临时目录中（而不是直接解压到 Python 的 site-packages 中）。

  • 运行 setup.py install 2。在 Windows 系统下，双击 install.py 可能就足够了。

2

setup.py 需要 setuptools 包。手动安装请参见 为 Docutils 的开发进行设置 中的选项。

可选步骤：

  • 运行测试套件

  • 转换文档

下面是特定于操作系统的安装说明。

GNU/Linux, BSDs, Unix, Mac OS X 等。¶

  1. 打开一个 shell。

  2. 进入包含 setup.py 的目录

    cd <archive_directory_path>

  3. 安装包（您可能需要 root 权限来完成此步骤）:

    su
(enter admin password)
python setup.py install

    如果 python 可执行文件不在你的路径上，你必须指定完整的路径，比如 /usr/local/bin/python。

    要安装特定的 Python 版本，请在 setup 调用中使用该版本，例如

    python3.7 setup.py install

    要针对不同的 Python 版本进行安装，请对每个所需的版本重复步骤3。最后安装的版本将在 rst2*.py 包装器脚本的 shebang 行 中使用。

Windows¶

只要双击 install.py。如果这不起作用，尝试以下方法：

  1. 打开一个 DOS 盒子（命令 Shell, MS-DOS 提示符，或者现在人们叫它什么）。

  2. 进入通过扩展归档文件创建的目录

    cd <archive_directory_path>

  3. 安装包

    <path_to_python.exe>\python setup.py install

    To install for a specific python version, specify the Python executable for this version.

    要针对不同的 Python 版本进行安装，请对每个所需的版本重复步骤3。

用法¶

在未打包的“tools”子目录中有许多前端工具。在 Unix 下安装将复制放在 PATH 中。您可能想从“rst2html.py”前端工具开始。大多数工具最多接受两个参数，源路径和目标路径，STDIN 和 STDOUT 是默认值。在前端工具中使用 --help 选项获取选项和参数的详细信息。阅读 Docutils Front-End Tools 的全部文档。

软件包的模块在不断地增长和发展。docutils.statemachine 模块是可以独立使用的。它包含大量的内联文档（当然是以 reStructuredText 格式）。

欢迎贡献！

项目文件和目录¶

  • README：你当前正在阅读的

  • COPYING：公共域（Public Domain，简称 PD）奉献和非公共域文件的版权细节（大多数是 PD）。

  • FAQ：常见问题（带答案！）

  • RELEASE-NOTES：最近版本中主要变化的摘要。

  • HISTORY：详细的更改日志，用于当前和所有以前的项目版本。

  • BUGS：已知的错误，以及如何报告错误。

  • THANKS：贡献者名单。

  • setup.py：安装脚本。请参阅下面的“安装”。

  • install.py：快速和纯净的安装脚本。只要运行它。但是，对于任何类型的定制或帮助，都必须使用 setup.py。

  • docutils/：项目源目录，作为 Python 包安装。

  • docs/：项目文档目录。请阅读 docs/index 获取概述。

  • docs/user：项目用户文档目录。包括下列文件：

    • docs/user/tools：Docutils 前端工具

    • docs/user/latex：Docutils LaTeX 写作

    • docs/user/rst/quickstart：ReStructuredText 快速上手

    • docs/user/rst/quickref.html: ReStructuredText 快速上手（仅针对 HTML）

  • docs/ref：项目参考目录。docs/ref/rst/restructuredtext.txt 是 reStructuredText 参考文献。

  • licenses：包含非公共域文件的许可文件副本的目录。

  • tools：Docutils 前端工具的目录。阅读 docs/user/tools.txt 了解更多信息。

  • test：单元测试。不需要使用该软件，但如果你打算修改它，它是非常有用的。阅读下面的 运行测试套件。

转换文档¶

解压并安装 Docutils 包后，以下 shell 命令将为所有包含的文档生成 HTML

cd <archive_directory_path>/tools
./buildhtml.py ../

Windows 系统，输入

cd <archive_directory_path>\tools
python buildhtml.py ..

对于 snapshots，<archive_directory_path> 的最终目录名是“docutils”。对于正式版本，该目录可能被称为“docutils-X.Y.Z”，这里 “X.Y.Z” 是发布版本。另外

cd <archive_directory_path>
tools/buildhtml.py --config=tools/docutils.conf          (Unix)
python tools\buildhtml.py --config=tools\docutils.conf   (Windows)

有些文件可能会生成系统消息（警告和错误）。docs/user/rst/demo.txt 文件（在归档目录下）包含5个故意错误。（他们测试错误报告机制！）

运行测试套件¶

测试套件被记录在 Docutils Testing （docs/dev/testing.txt）中。

要运行整个测试套件，打开一个 shell 并使用以下命令

cd <archive_directory_path>/test
./alltests.py

Windows 中，输入:

cd <archive_directory_path>\test
python alltests.py

你会看到一条长长的周期线，每个测试一个周期，然后是一个总结，像这样

Ran 1111 tests in 24.653s

OK
Elapsed time: 26.189 seconds

测试的数量将随着时间的推移而增长，报告的时间将取决于运行测试的计算机。这两个时间之间的差异表示设置测试(导入模块、创建数据结构等)所需的时间。

如果任何测试失败，请 打开错误报告 或 发送电子邮件 （请参阅 Bugs ）。请包括所有相关输出、有关操作系统、Python 版本和 Docutils 版本的信息。要查看 Docutils 版本，使用 rst2* 前端或 tools/quicktest.py 中的 --version 选项，例如

cd ../tools
./quicktest.py --version

Windows 用户键入这些命令

cd ..\tools
python quicktest.py --version
